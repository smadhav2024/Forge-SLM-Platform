2. The 6-Layer Fine-Tuning Pipeline (The Core Engine)

This pipeline acts as a deterministic assembly line. All files uploaded to /api/datasets/process must pass through these layers sequentially.

Layer 1: Universal File Ingestion

Your backend must abstract away the physical file format into a unified internal representation (like a list of dictionaries or a standard Pandas DataFrame).

  • CSVs/Excel (pandas): Read the file. If multiple columns exist without clear headers, dynamically serialize the row into a text block (e.g., "Column1: Value | Column2: Value").
  • PDFs/Word (PyMuPDF/fitz, python-docx): Extract raw UTF-8 text page-by-page. Discard all images, tables, and binary layout wrappers. Strip excessive whitespace and null bytes.

Layer 2: Automated Schema Detection (The Sorting Hat)

Do not invoke an LLM here. Use fast Python heuristics (Regex and Pandas column profiling) to determine the dataset's native structure.

  • Instruction Dataset: Detect if column headers match regex patterns like (?i)^(question|prompt|input|instruction)$ and (?i)^(answer|response|output|completion)$.
  • Chat Log (ChatML): Scan text for recurring linear anchors like [User]: and [Assistant]:.
  • Raw Text (Fallback): If no structural markers exist (e.g., a standard PDF manual), flag the dataset as unstructured_prose.

Layer 3: The Automated Cleaning Pipeline (The Car Wash)

Implement strict programmatic filters to sanitize the dataset.

  • Fuzzy Deduplication: Implement MinHash Locality-Sensitive Hashing (LSH) using a library like datasketch. If two rows are 85% similar or more, flag the duplicate.
  • Quality & Length Filters:
    • Drop rows with fewer than 20 characters.
    • Calculate the 95th percentile length of the dataset. Flag any row that is 4x longer than this average to prevent Out-Of-Memory (OOM) tensor crashes.
  • Gibberish Removal / Perplexity: Pass rows through a tiny, fast tokenizer/model reference (e.g., TinyLlama via HuggingFace transformers). If a row's perplexity is wildly above the dataset mean, flag it as corrupted binary/text.
  • PII Scrubbing: Use regex or Microsoft's presidio-analyzer to redact Emails, Phone Numbers, and IP addresses with [REDACTED].

Crucial Rule: Layer 3 must execute Soft Deletes. Flagged rows are moved to a filtered_out_data.jsonl quarantine list, NOT permanently deleted.

Layer 4: Hybrid Task Formatting (Python + AI)

If Layer 2 detected unstructured_prose, execute this hybrid workflow to synthesize training pairs.

  • The Python Cleaver: Use langchain.text_splitter.RecursiveCharacterTextSplitter. Set chunk_size=500 (words/tokens) and chunk_overlap=50. This preserves semantic boundaries across paragraphs.
  • The AI Scalpel: Trigger an async worker running a local 1B model (e.g., Gemma 3 1B IT). Loop over every chunk. Prompt the model to read the chunk and generate explicit Q&A pairs (Detailed in Section 3).

Layer 5: User Review Dashboard & The "Escape Hatches"

Before returning success to the frontend, compile a summary. The frontend requires specific endpoints to allow user intervention before triggering the GPU training script.

Required API Contracts to Build:

  1. GET /api/datasets/{id}/summary: Returns JSON with total_rows, rows_removed, duplicate_count, and a preview_samples array of 5 randomly selected final pairs.
  2. GET /api/datasets/{id}/quarantine: Returns paginated rows from the quarantine file (filtered_out_data.jsonl) with their rejection reason.
  3. PUT /api/datasets/{id}/restore: Accepts a list of row IDs. Moves them from the quarantine file back into the primary training dataset.
  4. POST /api/datasets/{id}/reprocess: Accepts JSON payload with slider values (e.g., {"dedup_threshold": 0.90, "chunk_size": 1000}). Re-runs Layers 3 & 4 with these overrides.
  5. PUT /api/datasets/{id}/edit-pair: Allows manual inline text updates to a specific AI-generated Q&A pair.
  6. DELETE /api/datasets/{id}: Aborts the process, wipes the temporary .jsonl files, and clears DB records.

Layer 6: Dynamic LoRA Configuration

Write a Python utility function that evaluates the final row count of the approved .jsonl file and outputs a configuration dictionary for the HuggingFace SFTTrainer:

  • Dataset < 1,000 rows: rank (r) = 4, lora_alpha = 8. Target modules: standard attention layers.
  • Dataset 1,000 to 10,000 rows: rank (r) = 8, lora_alpha = 16.
  • Dataset > 10,000 rows: rank (r) = 16, lora_alpha = 32. Target modules expanded to all linear layers to capture complex domain features.
  • Loss Masking: Ensure DataCollators are configured to mask the instruction/prompt tokens so gradients are only computed on the response tokens.

3. Deep Dive: Synthetic Data Generation Subsystem (For Layer 4)

To prevent the local background AI (The "AI Scalpel") from hallucinating or outputting conversational filler (e.g., "Here are your questions:"), you must enforce strict structured outputs during generation.

Implementation Steps:

  1. Define the Pydantic Schema: 
    from pydantic import BaseModel
from typing import List

class QAPair(BaseModel):
    instruction: str
    response: str

class QAExtraction(BaseModel):
    pairs: List[QAPair]
  2. System Prompting: Inject this exact system prompt: "You are an expert data engineer. Extract exactly 3 factual Question-and-Answer pairs based strictly on the provided text chunk. Do not invent information. Output only valid JSON."
  3. Guided Decoding: Use a library like outlines, instructor, or vLLM's native guided decoding functionality. Pass the QAExtraction Pydantic model directly into the generation parameters. This physically forces the model to adhere strictly to your JSON schema.
  4. Compilation: Use Python's json.loads() on the output, validate it against the Pydantic model, and use jsonlines to append it to the master .jsonl file.
